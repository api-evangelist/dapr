Creating platforms on top of Kubernetes is not an easy task, but some tools will make your life easier when doing so. There seems to be a trend in the Cloud-Native space where the Kubernetes API is extended to serve as the Platform interface for development teams to request the resources they need to do their work. We have seen how this can be achieved with tools like Crossplane in this other blog post, but this time we will look into Kratix, a project born to help platform builders build flexible platforms.
